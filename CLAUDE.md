# CLAUDE.md
日本語で回答およびCLAUDE.mdを記述すること
このファイルは、リポジトリで作業する Claude Code (claude.ai/code) へのガイダンスを提供します。

## 概要

グラッチェコイン（GrazieCoin）のオンラインマニュアル用 MkDocs サイトです。すべてのコンテンツは Markdown で記述され、Material for MkDocs でレンダリングされます。

## コマンド

| スクリプト | 用途 |
|---|---|
| `./setup.sh` | 仮想環境の作成とパッケージインストール（初回のみ） |
| `./serve.sh` | ローカルプレビュー（ライブリロード付き） |
| `./build.sh` | 静的 HTML のビルド |

各スクリプトは `.venv` を直接参照するため、activate 不要で実行できます。初回は必ず `./setup.sh` を先に実行してください。

`release` ブランチへの PR マージ時に、GitHub Actions で GitHub Pages へ自動デプロイされます。

## アーキテクチャ

- **`mkdocs.yml`** — サイト設定（ナビゲーション構造・テーマ・拡張機能）。このファイルは変更しない。
- **`docs/`** — すべての Markdown コンテンツページ
  - **`一般機能/`** — 一般ユーザマニュアルページ
  - **`管理者機能/`** — 管理者ユーザマニュアルページ
  - **`howto/`** — クイックスタートガイド
  - **`app/`** — スマホアプリマニュアル
  - **`releasenote/`** — バージョン別リリースノート
  - **`images/`** — 機能エリアごとに整理されたスクリーンショット
  - **`libs/`** — 画像ギャラリー用 Lightbox JS/CSS
- **`template.md`** — 新規マニュアルページの参照テンプレート

## ファイル管理ルール

- コマンド・スクリプト・環境構成・アーキテクチャに変更を加えた場合は、**必ず README.md を最新の状態に更新**する。

## コンテンツルール

すべての出力は**日本語**で「です・ます調」で記述します。

### ページ構成（一般ユーザ・管理者マニュアルページ）
1. 機能タイトル（`#` 見出し）
2. Lightbox アンカー形式のスクリーンショット（正確な HTML は `template.md` を参照）
3. UI 項目テーブル（列: `#` / 項目名 / 必須 / 説明）
4. 操作方法セクション（Scribe の埋め込み iframe、`src=""` はプレースホルダーのまま）

### Markdown 書式
- 見出しは `#` ～ `###` まで（最大 3 階層）
- セクション間には空行を 1 行入れる
- 箇条書きは `-` または `*`、手順には番号付きリスト（`1. 2. 3.`）を使う
- テーブルは Markdown テーブル形式で整形し、中央揃えには `:---:` を使う

### 禁止事項
- 生成できるのは Markdown ファイルのみ（JS・CSS・画像・バイナリ不可）
- `mkdocs.yml` は変更しない
- 推測や個人的な意見は書かない
- 未完成のセクションには `※この部分は後で補足` をプレースホルダーとして使う
